# ECE4076_WALL_E

Lab 1 Q10 Discussion

The aim of this lab was to use a mixture of convolution, filtering techniques and corner detection so that the minutia of fingerprints in raw images can be digitally identified.

Steps 1 and 2 focused on preprocessing of the image. Importing the image, cropping it, greyscaling it, and equalising the image. Equalising the image made clear improvement in ridge and valley contrast, which resulted in a much sharper image output after equalisation.

Gaussian blurring was implemented in step 4, where a kernel size of 5 and σ = 3.0 suppressed high-frequency noise before applying the Sobel filter in the x and y directions. This helped to reduce the noise seen in the latent fingerprint images. It was found that smaller sigma of ~1.0 kept too much noise, causing the orientation map to appear chaotic with no identifiable ridge structure. A larger σ over-smoothed ridges into each other. A Sobel kernel size of 5 was able to cover gradient at ridges, without blurring across adjacent ridges.
Step 5 computed the block-wise average orientations as well as an orientation histogram, showing the distribution of ridge orientations across the image and which orientations were most prevalent. A block size of d = 12 pixels was chosen because it is half the typical ridge spacing visible in the images, providing enough resolution to capture local orientation changes without being influenced by local noise.

Step 6 applied Gabor filtering. This step appeared to have the most effect on the final threshold image outputs and therefore how well the corner detectors could accurately detect minutia. A kernel size of 31, σ = 3.9, and frequency = 0.40 were chosen after a lot of tuning. Lower frequencies produced broad, blurred responses, while higher frequencies introduced artefacts between ridges. The σ value of 3.9 matched the approximate ridge width in the equalised images. The larger kernel size gave the filter more spatial context to match ridge structure across its full extent.
Step 7 combined the filtered responses and applied a thresholded. The threshold was tuned to 139 after testing, as it kept enough detail in the ridges, while discarding background noise. Compared to thresholding the equalised image directly at the same value, the Gabor-enhanced result produced much cleaner ridge segments with practically no background detection.
Steps 8 covered minutiae detection, where Shi-Tomasi corner detection was used. Parameters of max 800 corners, quality level 0.01, minimum distance 13 px, and block size 9 were used to avoid over-clustering of detected corners, while still detecting fine ridge features. The enhanced binary image produced more accurate minutiae than the original, with very few false detections.

The most impactful parameter across the whole pipeline was gabor_freq = 0.40, which needs to match the spatial frequency of the ridges. Adjusting it by even 0.05 visibly degraded the enhancement, regardless of any other settings. The binary threshold in step 7 was also a very influential parameter, as values above 160 broke ridge continuity and created false minutiae. Threshold values below 120 left noise blobs that drowned out real detections. At the gradient stage, gauss_sigma had a big effect since too much smoothing merged adjacent ridges and corrupted the orientation map feeding into Gabor filter selection.

The main limitation of this approach is that corner detection is an approximate stand-in for true minutiae, since ridge endings have different local structures that are not told apart by the detector. Gabor filtering also assumes locally uniform ridge spacing and orientation, which does not work in smudged regions. Finally, all parameters were tuned manually for this image pair, meaning a fingerprint with different ridge density would need gabor_freq and threshold values adjusted, which reduces how broadly the pipeline can be applied.
